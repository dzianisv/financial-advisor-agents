## Vault collateral risk learnings (2026-06-20)

Fluid-lite (Instadapp) is a leveraged loop vault — NOT a blue-chip lending pool. It converts USDC → synthetic yield stablecoins (sUSDe/syrupUSDC/syrupUSDT/sUSDai), loops them as collateral on Fluid/Aave/Euler to amplify yield. Looks like a simple USDC deposit but has synthetic stablecoin + leverage risk. DeFiLlama pool `488f06db`. APY ~8.44% = leverage premium, not borrower demand. Had $21M bad debt event March 2026. Classify as C3/C4.

Aave-v3 USDC Umbrella (ETH, pool `6f00d46b`) = confirmed C1 at 7.13% APY, $56M TVL — best single recommendation for USDC on Ethereum mainnet.

Collateral risk classification C1-C4 now in overview-crypto-portfolio SKILL.md. Always match tiers before recommending pool upgrades. High APY on a "USDC" pool is a red flag, not a free lunch — check what the pool lends against.

## research-market workflow bugs fixed (2026-06-20)

### Bug 1: ARGS key mismatch (PR #46, merged)
- `ARGS.question` was used in workflow but callers passed `ARGS.query` — every equities run silently defaulted to hardcoded BTC query
- Fix: normalize both keys + remove hardcoded crypto default
- Lesson: always validate ARGS field names match between workflow definition and callers; hardcoded fallbacks mask missing input

### Bug 2: Monolithic phase timeout (PR #47)
- Old architecture ran all 13 tickers in one monolithic gather/consolidate phase → stalled at 50+ min
- Fix: per-asset pipeline — each ticker gets dedicated gather/consolidate/panel agents spawned in parallel via pipeline(ASSETS)
- Lesson: fan out per-asset in parallel from the start; never serialize ticker loops in a single agent
